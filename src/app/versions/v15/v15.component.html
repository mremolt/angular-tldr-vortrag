Neuigkeiten in Angular V15

Standalone Components (stable)

Mit Angular 15 wurde die Standalone API als stabil gekennzeichnet, also für alle Entwickler freigegeben. Als Ausblick, mit V17 wurde Standalone der Standard für neue Projekte (ng new).

Image Directive

Die Angular Image Directive, genauer "NgOptimizedImage" ist ein Ersatz für das src-Attribut bei img-Tags. Der Vorteil gegenüber dem normalen "src" ist, dass so viele Optimierungen in Richtung Web Vitals und best practices aktiviert werden.

  • Verwaltet "fetchpriority" und lazy loading von Bildern.
  • Verwendet "srcset" für verschiedene Bildgrößen (Thema mobile).
  • Einfache Anbindung an CDNs.
  • Unterstützt beim Generieren von Anwendungen mit SSR-Unterstützung.

Generell, gerade wer in einem öffentlich sichtbaren Projekt arbeitet, wo Themen wie Web Vitals oder SEO relevanter sind, ist die Image Directive ein einfacher "Win" ohne großartigen Mehraufwand oder Nachteile.

Router Guards als Funktionen

Angular Material: Umstieg auf MDC Web Components

Die Komponenten-Bibliothek "@angular/material" setzt nun auf dem Material-Schwesterprojekt MDC (Material Components for Web) auf. In den Angular Components werden nun also native Web Components verwendet, die eigentlichen Angular Komponenten sind größtenteils nur Container, die Angular-spezifische Funktionen nachrüsten.

Wichtig: Für alle mit legacy Projekten, die demnächst auf Material >= 15 aktualisieren müssen. Das ganze Custom CSS in eurer App, mit dem ihr Material an das Kunden-UX angepasst habt, wird nicht mehr funktionieren.
